# 🧠 Debugging Playbook

A personal knowledge base of bugs, fixes, and engineering lessons learned while building real-world systems.

---

## 📌 Purpose

This repository exists to:

- Document non-trivial bugs and their solutions  
- Avoid repeating the same mistakes  
- Build a structured engineering mindset  
- Track real learning through practical problems  

---

## 🛠️ Structure

```txt
KB-debugging-playbook
├── bugs/       # Specific bugs and how they were solved
├── backend/    # Backend concepts and implementation notes
└── infra/      # Environment, Docker, deployment issues
```

---

## 🧩 Entry Format

Each note follows a simple structure:

- **Context** – What I was trying to do  
- **Problem** – What went wrong  
- **Root Cause** – Why it actually happened  
- **Solution** – How it was fixed  
- **Takeaway** – What I learned  

---

## ⚡ Philosophy

> If it took time to debug, it’s worth documenting.

This is not a diary.  
Only meaningful problems and insights are recorded.

---

## 🚀 Why this matters

Debugging is a core engineering skill.

This repository reflects:

- Problem-solving ability  
- Technical growth  
- Attention to detail
